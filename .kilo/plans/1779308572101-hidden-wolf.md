# Walkthrough & Quote Appointment Calendar System

## Overview

This document outlines a comprehensive system for scheduling and tracking **in-person walkthroughs and quote appointments** for the cleaning company. This is distinct from regular cleaning job scheduling — this system manages **sales appointments** where representatives visit potential clients to conduct property assessments and provide quotes.

---

## 1. Problem Statement

Currently, the system has:
- **BookingCalendar**: For customers to book cleaning services online
- **JobsManager**: Kanban-style job scheduling for actual cleaning work
- **LeadsList**: CRM for tracking lead status (New → Contacted → Converted)

**What's missing**: A dedicated system to log promises made during follow-up calls, schedule in-person walkthroughs, and track quote appointments without scheduling conflicts.

---

## 2. Recommended Solution Architecture

### 2.1 Database Schema (PostgreSQL)

**New Table: `walkthrough_appointments`**

```sql
CREATE TABLE walkthrough_appointments (
  id SERIAL PRIMARY KEY,
  lead_id INTEGER REFERENCES leads(id) ON DELETE CASCADE,
  scheduled_date DATE NOT NULL,
  scheduled_time TIME NOT NULL,
  duration_minutes INTEGER DEFAULT 60,
  address TEXT,
  service_type VARCHAR(100),          -- e.g., "Commercial Walkthrough", "Quote Visit"
  status VARCHAR(50) DEFAULT 'Scheduled',
  -- Statuses: Scheduled, Confirmed, Completed, Cancelled, No-Show, Converted
  notes TEXT,
  outcome TEXT,                        -- Quote provided, Follow-up needed, etc.
  quote_amount DECIMAL(10,2),
  follow_up_date DATE,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  created_by VARCHAR(100)
);

CREATE INDEX idx_walkthrough_date ON walkthrough_appointments(scheduled_date);
CREATE INDEX idx_walkthrough_lead ON walkthrough_appointments(lead_id);
```

**Note**: Shared calendar model - no per-rep assignment.

### 2.2 Data Model (Frontend)

```typescript
interface WalkthroughAppointment {
  id: number;
  lead_id: number;
  lead_name: string;
  lead_phone: string;
  scheduled_date: string;  // YYYY-MM-DD
  scheduled_time: string;  // HH:MM
  duration_minutes: number;
  address: string;
  service_type: 'Commercial Walkthrough' | 'Quote Visit' | 'Follow-up Visit' | 'Initial Assessment';
  status: 'Scheduled' | 'Confirmed' | 'Completed' | 'Cancelled' | 'No-Show' | 'Converted';
  notes: string;
  outcome: string;
  quote_amount: number | null;
  follow_up_date: string | null;
  created_at: string;
  created_by: string;
}
```

**Note**: Shared calendar model - no per-rep assignment. All team members see the same calendar.

## 3. Key Features

### 3.1 Calendar View
- **Day/Week/Month views** to see all walkthrough appointments
- **Conflict detection**: Visual warning when double-booking is attempted
- **Color-coded by status**: Scheduled (blue), Confirmed (green), Completed (purple), Cancelled (red), No-Show (orange)
- **Lead info integration**: Click appointment to see full lead details

### 3.2 Scheduling Interface
- **Quick schedule from Lead**: Button on lead profile to schedule walkthrough
- **Time slot picker**: Pre-defined slots (9 AM, 10 AM, 11 AM, 1 PM, 2 PM, 3 PM, 4 PM)
- **Duration settings**: 30min, 60min, 90min, 2hr options
- **Address field**: Critical for walkthroughs

### 3.3 Conflict Prevention
- **Real-time availability check** before confirming
- **Visual indicator of conflicting appointments**
- **Warning modal when scheduling over existing appointment
- **Buffer time option**: 15min between appointments (optional)
- Shared calendar - conflict = same date/time already booked

### 3.4 Appointment Lifecycle
1. **Scheduled** → Initial booking from follow-up call
2. **Confirmed** → Client confirmed attendance
3. **Completed** → Walkthrough finished
4. **Converted** → Lead converted to customer
5. **Cancelled** → Appointment cancelled
6. **No-Show** → Client didn't show

### 3.5 Follow-up Tracking
- **Post-appointment outcome capture**
- **Quote amount logging**
- **Follow-up date scheduling**
- **Automatic lead status update (e.g., → "Quote Provided")

## 8. Technical Recommendations

### Why This Approach
1. **PostgreSQL** is ideal for scheduling data with date range queries
2. **Server-side conflict detection** prevents race conditions
3. **Extending existing context** maintains single source of truth
4. **Kanban-like status flow** matches existing UX patterns

### Alternative Tools Considered
- **Google Calendar API**: More complex setup, external dependency
- **FullCalendar library**: Could integrate, but custom calendar already in JobsManager
- **Third-party scheduling SaaS**: Additional cost, less control

## Implementation Decisions

- **Auto-update lead status**: YES - Walkthroughs will automatically transition lead status (e.g., "Walkthrough Scheduled" → "Walkthrough Completed" → "Quote Provided" → "Converted")
- **Team assignment**: NO - Shared calendar, no per-rep tracking needed
- **Reminders**: NO - No automated reminders required

## Status Flow for Leads

Based on walkthrough events:
1. Lead created → Status: "New"
2. Walkthrough scheduled → Status: "Walkthrough Scheduled"
3. Walkthrough completed → Status: "Quote Provided"
4. Quote accepted → Status: "Converted"
