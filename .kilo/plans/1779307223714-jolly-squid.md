# Dashboard Autonomy Analysis - 360 Cleaning Co.

## Executive Summary
Current dashboard has solid foundations (Leads CRM, Jobs, Crew, Invoices, Marketing, Reports) but operates in **manual intervention mode**. The system cannot run autonomously because core operational workflows lack automation triggers.

---

## PART 1: CRITICAL MISSING FEATURES FOR AUTONOMOUS OPERATION

### 1.1 Automated Lead Follow-up System
**Problem:** Leads sit untouched - no automatic reminders when leads go cold.

**Missing:**
- Time-based triggers for unresponsive leads (e.g., "no response in 3 days")
- Automatic SMS/email follow-up sequences
- Escalation rules (New → Contacted → Converted happens manually)
- "Stale lead" alerts in dashboard

**Impact:** High - 40-60% of leads convert if followed up within 5 minutes; current system relies on manual tracking.

---

### 1.2 Recurring Job Auto-Generation
**Problem:** Recurring clients exist but jobs must be manually scheduled each cycle.

**Missing:**
- Background job that reads recurring client next_date and auto-creates jobs
- Automatic status check: "If next_date <= today AND status=active, create job"
- Next date recalculation after job completion

**Impact:** Critical - This single feature eliminates daily manual scheduling work.

---

### 1.3 Intelligent Crew Assignment
**Problem:** Jobs are created without automatic crew matching.

**Missing:**
- Crew-to-job matching based on:
  - Geographic zone/route optimization
  - Crew availability status
  - Skills/certifications (e.g., "deep clean certified")
  - Current workload balancing
- Auto-assignment when job is created
- Manual override capability

**Impact:** High - Reduces 15-20 minutes of scheduling work per day.

---

### 1.4 Overdue Payment Automation
**Problem:** Invoice status must be manually changed; no automatic reminders.

**Missing:**
- Cron job to check invoice due dates daily
- Automatic status change: pending → overdue when past due_date
- Automated payment reminder emails/SMS at 1 day, 3 days, 7 days overdue
- Late fee calculation rules

**Impact:** Medium - Improves cash flow without manual tracking.

---

### 1.5 Customer Status Notifications
**Problem:** No automated notifications to customers about their job status.

**Missing:**
- Automatic SMS/email when:
  - Lead is received (confirmation)
  - Quote is ready
  - Job is scheduled
  - Job is completed
  - Payment is due
- Customer-facing web portal to track job status

**Impact:** Medium - Reduces "Where is my cleaner?" support calls by 80%.

## PART 3: AUTOMATED WORKFLOWS NEEDED

### 3.1 Daily Operations Workflow
```
6:00 AM  → System checks tomorrow's jobs
         → Validates crew availability
         → Sends SMS reminders to customers
         → Alerts if understaffed for demand

7:00 AM  → System assigns crews to new jobs
         → Optimizes routes by geographic zone
         → Sends crew their daily schedule

8:00 AM  → Monitor job starts (manual check-in via crew app)
         → If no check-in, send alert

4:00 PM  → Check completed jobs
         → Auto-generate invoices for completed jobs
         → Update recurring client next_date

6:00 PM  → Send follow-up SMS: "How did we do?" with review link
```

### 3.2 Lead Nurturing Workflow
```
Lead enters system
  → Send welcome SMS within 2 minutes
  → If no response in 2 hours → email follow-up
  → If no response in 24 hours → different SMS template
  → If no response in 48 hours → mark as "stale" (badge in dashboard)
  → If no response in 72 hours → alert admin for manual outreach
```

### 3.3 Recurring Client Workflow
```
Every night at midnight
  → Check all recurring clients where next_date <= today AND status=active
  → Create job for each matching client
  → Set job date = today
  → Assign crew based on client's historical crew or zone
  → Update recurring client's next_date based on frequency
  → Send SMS to customer: "Your cleaning is scheduled for today"
```

- Inbound SMS handling (keywords like "STOP", "RESCHEDULE", "CONFIRM")

### 4.6 Integration Ecosystem

- **Call tracking**: Log all phone calls, transcription
- **Review platforms**: Auto-request Google/Facebook reviews
- **Google Ads/Facebook Pixel**: Lead source tracking integration

### 4.7 Inventory & Supply Management
- Track cleaning supplies per job type
- Auto-reorder when stock low
- Supply cost per job calculation for true margin analysis
- Equipment maintenance scheduling

### 4.8 Commission & Payroll System
- Calculate crew commissions per job

- Generate payroll reports

## IMPLEMENTATION APPROACH

User has confirmed: **Implement All Phases**

### Implementation Order:
1. **Database** - Add tables for: `automated_workflows`, `job_schedules`, `sms_sequences`, `email_sequences`, `crew_zones`, `audit_log`
2. **Backend API** - New endpoints for workflow automation, scheduling jobs, crew assignment logic
3. **Frontend** - Add workflow configuration UI, enhanced reports, notification center

### Key Technical Changes:

#### Database Schema Additions
```sql
-- Automated workflows table
CREATE TABLE automated_workflows (
  id SERIAL PRIMARY KEY,
  name VARCHAR(255),
  trigger_type VARCHAR(50), -- 'time', 'event', 'schedule'
  trigger_config JSONB,
  action_type VARCHAR(50), -- 'sms', 'email', 'create_job', 'assign_crew'
  action_config JSONB,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT NOW()
);

-- SMS sequences for lead nurturing
CREATE TABLE sms_sequences (
  id SERIAL PRIMARY KEY,
  lead_status VARCHAR(50), -- 'New', 'Contacted'
  step_order INT,
  delay_hours INT,
  template_id VARCHAR(50),
  is_active BOOLEAN DEFAULT true
);

-- Crew service zones for assignment
CREATE TABLE crew_zones (
  id SERIAL PRIMARY KEY,
  crew_id INT REFERENCES crew_members(id),
  zone_name VARCHAR(100),
  zip_codes TEXT[]
);
```

#### Backend Changes (server/index.js)
- Add `/api/workflows` CRUD endpoints
- Add `/api/jobs/auto-generate` endpoint
- Add `/api/crew/assign` endpoint with optimization logic
- Add cron-like job processor (can use node-cron or similar)
- Add `/api/notifications/send` endpoint

#### Frontend Changes
- New `WorkflowAutomation.jsx` component
- Enhanced `Reports.jsx` with charts (Chart.js or Recharts)
- New `NotificationCenter.jsx` component
- Update `AdminDashboard.jsx` with automation status widgets
