### Phase 4: Backend SMS Automation Logic

**4.1 New `processSmsSequences` Function**
- Hourly cron job (already exists for other processes)
- For each lead with status "New", "Contacted", or "Stale":
  1. Get the appropriate service-specific sequence
  2. Find the next step that should be sent (based on last_sent_at + delay_hours)
  3. Replace placeholders: `{{name}}`, `{{service}}`, `{{county}}`, `{{estimated_price}}`
  4. Send SMS via QUO API
  5. Update sequence state (last_sequence_step, last_sent_at)

**4.2 New Tables for Sequence Tracking**
```sql
CREATE TABLE IF NOT EXISTS lead_sms_sequence_state (
  lead_id INT REFERENCES leads(id) UNIQUE,
  service_type VARCHAR(50),
  last_sequence_step INT DEFAULT 0,
  last_sent_at TIMESTAMP,
  completed BOOLEAN DEFAULT FALSE,
  completed_at TIMESTAMP
);
```

**4.3 Automation Flow**
- **Hourly cron** checks for leads needing follow-up:
  1. Find leads with status "New", "Contacted", or "Stale" where:
     - `completed = FALSE`
     - `last_sent_at + (delay_hours * interval) < now()`
     - Haven't reached the final sequence step
  2. Send the next SMS in the sequence
  3. Update `last_sequence_step` and `last_sent_at`
  4. If final step sent, mark `completed = TRUE`

**4.4 Mark Leads as Stale**
- Cron job runs every hour
- Finds leads where:
  - Status IN ('New', 'Contacted')
  - `updated_at < now() - interval '72 hours'`
  - Not already marked "Stale"
- Updates status to "Stale"
- Triggers Stale re-engagement sequence

### Phase 5: Enhanced Marketing Center UI

**5.1 Marketing Center at `/admin/marketing`**

Four main tabs:

**Dashboard Tab**:
- SMS campaign stats: sent, delivered, failed
- Conversion metrics: leads → booked jobs
- Recent activity feed
- Quick actions: send manual campaign

**SMS Campaigns Tab**:
- **Pre-built templates by service type** (editable):
  - Residential, Commercial, Deep Cleaning, Move In/Out, Post-Construction
  - Re-engagement templates for stale leads
  - Win-back templates for converted customers
- **Template preview** with live placeholder substitution
- **Send options**:
  - All leads
  - Filter by service type
  - Filter by status (New, Contacted, Stale, Converted)
  - Selected individual leads
- **Personalization tokens**: `{{name}}`, `{{service}}`, `{{property_size}}`, `{{county}}`, `{{estimated_price}}`
- **One-time campaigns** vs automated sequences

**Sequences Tab** (View Only for Now):
- View the automation sequences by service type
- See step number, delay, message preview, trigger status
- Ability to enable/disable specific sequences

**Contacts Tab**:
- All leads with SMS history
- Shows: name, phone, service, status, last SMS sent, sequence step
- Manual SMS send to individual lead
- View full SMS conversation thread

**5.2 New API Endpoints**
- `GET /api/marketing/stats` - Dashboard statistics
- `GET /api/marketing/leads` - Leads with SMS history
- `POST /api/marketing/send-single` - Manual SMS to single lead
- `POST /api/marketing/send-campaign` - Send campaign to filtered leads
- `GET /api/service-sms-sequences` - Get sequences by service type
- `PUT /api/service-sms-sequences/:id` - Update a sequence step
